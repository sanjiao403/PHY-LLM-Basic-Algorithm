## 摘要

本文提出了 **rStar-Math**，展示了**小语言模型（SLMs）**可以在不依赖更优越模型蒸馏的情况下，匹敌甚至超越 OpenAI o1 的数学推理能力。

rStar-Math 通过 **Monte Carlo Tree Search (MCTS)** 实现"深度思考"，其中数学策略 SLM 在基于 SLM 的过程奖励模型引导下执行测试时搜索。

rStar-Math 引入了三项创新：

1. **新型代码增强的 CoT 数据合成方法**：通过广泛的 MCTS rollout 生成逐步验证的推理轨迹，用于训练策略 SLM
2. **新型过程奖励模型训练方法**：避免朴素步骤级分数标注，产生更有效的过程偏好模型（PPM）
3. **自我演化方案**：策略 SLM 和 PPM 从零开始构建并迭代演化以提升推理能力

通过 4 轮自演化，使用 747k 数学问题的数百万合成解决方案，rStar-Math 将 SLMs 的数学推理提升至最先进水平。在 MATH 基准上，将 Qwen2.5-Math-7B 从 58.8% 提升至 **90.0%**，Phi3-mini-3.8B 从 41.4% 提升至 **86.4%**，超越 o1-preview (+4.5% 和 +0.9%)。

## 核心贡献

1. **小模型超越 o1**：无需蒸馏，SLMs 通过自演化达到超越 o1-preview 的水平
2. **MCTS 深度思考**：使用蒙特卡洛树搜索进行测试时推理
3. **代码增强 CoT 数据合成**：生成逐步验证的推理轨迹
4. **过程偏好模型（PPM）**：避免朴素步骤级标注的新方法
5. **4 轮自演化**：策略 SLM 和 PPM 从零开始迭代提升

## 方法

### 整体架构

rStar-Math 包含两个核心组件：
1. **策略 SLM**：生成数学推理步骤
2. **过程偏好模型（PPM）**：评估中间步骤的质量

### 创新 1：代码增强的 CoT 数据合成

- 使用 MCTS rollout 生成大量候选推理轨迹
- 通过代码执行验证（Python 解释器）验证推理步骤的正确性
- 生成逐步验证的高质量推理数据

### 创新 2：过程偏好模型（PPM）训练

- 避免朴素的步骤级分数标注
- 基于 Q 值构建步骤级偏好对
- 通过 MCTS 的 Q 值自动筛选高质量步骤

### 创新 3：自我演化方案

4 轮迭代：
```
Round 1: 基础 SLM → MCTS rollout → 生成数据 → 训练 PPM
Round 2: PPM 引导 MCTS → 更高质量数据 → 更新 SLM 和 PPM
Round 3: 继续迭代...
Round 4: 最终模型
```

每轮迭代中：
- 策略 SLM 通过 MCTS 生成更多高质量数据
- PPM 使用新数据更新
- 两者相互促进，共同提升

### MCTS 深度思考

- 策略 SLM 作为策略模型生成推理步骤
- PPM 作为奖励模型评估步骤
- 通过 MCTS 搜索最优推理路径
- 支持 64 条轨迹的测试时搜索

## 关键洞察

1. **小模型的巨大潜力**：7B 甚至 3.8B 模型通过正确方法可以超越 o1-preview
2. **MCTS 的力量**：测试时搜索是提升推理能力的关键
3. **自演化的有效性**：无需外部蒸馏数据，模型可以自我提升
4. **PPM 优于朴素 PRM**：基于 Q 值的偏好模型比步骤级标注更有效

---

## 局限性与讨论

1. **计算成本**：MCTS 搜索需要大量测试时计算
2. **领域限制**：目前主要验证于数学推理
3. **PPM 训练复杂度**：Q 值过滤和偏好对构建需要精心设计
4. **可扩展性**：更大规模模型的自演化效果需要进一步验证

---

## 影响与意义

本文是小模型推理能力的里程碑：
- 首次证明小模型可以在不依赖蒸馏的情况下超越 o1
- MCTS 自演化框架为推理优化提供了新方向
- 展示了测试时计算替代预训练规模的可行性
- 对资源受限场景下的模型部署具有重要价值
