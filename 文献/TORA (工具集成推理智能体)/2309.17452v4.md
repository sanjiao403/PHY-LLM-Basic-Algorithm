## 摘要

本文提出了 **TORA**，一系列**工具集成推理智能体（Tool-integrated Reasoning Agents）**，旨在通过将自然语言推理与外部工具（如计算库和符号求解器）的使用无缝集成来解决具有挑战性的数学问题，从而融合语言的分析能力和工具的计算效率。

为了训练 TORA，研究团队在数学数据集上策划了交互式工具使用轨迹，应用模仿学习进行标注，并提出**输出空间塑造（Output Space Shaping）**来进一步优化模型的推理行为。结果表明，TORA 模型在所有规模上均显著优于开源模型，在 10 个数学推理数据集上平均提升 13%-19%。

值得注意的是，**TORA-7B 在竞赛级数据集 MATH 上达到 44.6%**，超过最佳开源模型 WizardMath-70B 达 22%。**TORA-CODE-34B 是首个在 MATH 上准确率超过 50% 的开源模型**，显著优于 GPT-4 的 CoT 结果。

## 核心贡献

1. **提出工具集成推理（TIR）格式**：交替使用自然语言推理和程序化外部工具调用
2. **构建交互式工具使用轨迹数据**：用于训练模型有效使用外部工具
3. **提出输出空间塑造**：进一步细化模型的推理行为
4. **显著的性能提升**：在所有规模上超越开源模型 13%-19%

## 方法

### 工具集成推理（TIR）格式

与传统的纯自然语言推理（CoT）或纯程序推理（PAL）不同，TIR 格式交替使用：

1. **推理步骤（Rationale）**：自然语言分析和规划
2. **程序调用（Program）**：编写代码调用外部工具（如 SymPy、NumPy）
3. **执行输出（Output）**：工具返回的计算结果
4. **最终推理**：基于执行输出得出最终答案

示例格式：
```
[推理] 我们需要解这个方程组...
[程序] import sympy; x, y = symbols('x y'); solve([Eq(x**2 + y**2, 7), Eq(x**3 + y**3, 10)], (x, y))
[输出] [(-5,), (1,), (4,)]
[推理] 因此，x+y 的可能值为 -5, 1, 4
```

### 训练流程

1. **数据策划**：
   - 在数学数据集上构建交互式工具使用轨迹
   - 结合人工标注和自动生成的解决方案

2. **监督微调（SFT）**：
   - 使用工具使用轨迹进行模仿学习
   - 训练模型学习何时以及如何调用工具

3. **输出空间塑造**：
   - 进一步优化模型的推理行为
   - 鼓励模型生成更结构化的推理-工具交替序列

## 实验结果

### MATH 基准测试

| 模型 | MATH 准确率 |
|------|------------|
| LLaMA-2-7B (Base) | 4.1% |
| WizardMath-70B | 22.7% |
| **TORA-7B** | **44.6%** |
| **TORA-13B** | **48.1%** |
| **TORA-70B** | **49.7%** |
| GPT-4 (CoT) | ~40% |
| GPT-4 (Code) | ~55% |

### GSM8K 基准测试

| 模型 | GSM8K 准确率 |
|------|-------------|
| LLaMA-2-7B (Base) | 13.3% |
| WizardMath-70B | 57.8% |
| **TORA-7B** | **72.6%** |
| **TORA-13B** | **75.8%** |
| **TORA-70B** | **84.3%** |
| GPT-4 (CoT) | ~81% |

### 10 个数学数据集上的平均提升

- TORA 在所有规模上平均提升 **13%-19%**
- 在竞赛级问题上的提升尤为显著

## 关键洞察

1. **工具集成的优势**：结合自然语言推理的灵活性和外部工具的精确计算
2. **推理-工具交替**：模型学会在适当的时候调用工具，而非一味依赖推理或计算
3. **规模效应**：更大的模型更好地学习工具使用模式
4. **开源模型的突破**：TORA 首次使开源模型在竞赛级数学问题上接近 GPT-4

---

## 局限性与讨论

1. **工具依赖**：需要外部执行环境来运行代码
2. **错误传播**：工具执行错误可能影响后续推理
3. **工具选择**：模型需要学习何时使用何种工具
4. **训练数据**：高质量的交互式工具使用数据难以获取

---

## 影响与意义

本文是工具增强推理的重要进展：
- 提出了实用的工具集成推理框架
- 显著缩小了开源模型与闭源模型之间的差距
- 为数学推理提供了新的范式
- 对科学计算和自动化推理有广泛应用前景
